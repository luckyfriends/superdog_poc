# 番茄钟桌面应用设计文档

**日期：** 2026-05-13  
**技术栈：** Electron + Vanilla JS  
**平台：** macOS（兼容 Windows/Linux）

---

## 一、概述

一个轻量级桌面番茄钟应用，现代渐变风格 UI，支持系统通知和托盘图标。核心目标是专注、干净、无干扰。

**功能范围：**
- 核心计时器（专注 25 分钟 / 短休息 5 分钟 / 长休息 15 分钟）
- 系统通知（每段结束时触发）
- 系统托盘图标（菜单栏显示剩余时间）

---

## 二、架构

### 进程模型

Electron 标准双进程架构：

| 进程 | 文件 | 职责 |
|------|------|------|
| 主进程 | `main.js` | 创建窗口、托盘图标、发送系统通知、监听 IPC |
| 渲染进程 | `renderer.js` | 计时器逻辑、UI 交互、状态管理 |
| 桥接层 | `preload.js` | 安全暴露主进程 API 给渲染进程（contextBridge） |

### 文件结构

```
tomato-clock/
├── main.js
├── preload.js
├── index.html
├── renderer.js
├── styles.css
├── assets/
│   ├── tray-icon.png         # 16×16 托盘图标（默认）
│   └── tray-icon-active.png  # 运行中状态图标
└── package.json
```

### IPC 接口（preload.js 暴露）

```js
window.api.notify(title, body)   // 触发系统通知
window.api.setTray(text)         // 更新托盘标题文字
```

## 四、计时器逻辑

### 状态机

```
IDLE → RUNNING ⇌ PAUSED
         ↓ (时间到)
      BREAK_SHORT / BREAK_LONG
         ↓ (时间到)
      RUNNING (下一段) 或 IDLE (循环完成)
```

### 会话流程

```
专注(25) → 短休息(5) → 专注(25) → 短休息(5)
→ 专注(25) → 短休息(5) → 专注(25) → 长休息(15)
→ 重新开始
```

### 计时精度

使用 `setInterval(fn, 1000)` 触发 UI 更新，实际剩余时间通过 `Date.now()` 计算（而非累加），避免系统休眠或标签页失焦导致漂移。

## 五、功能细节

### 系统通知

| 触发时机 | 标题 | 正文 |
|---------|------|------|
| 专注段结束 | 休息一下 🎉 | 完成第 N 个番茄，休息 5 分钟 |
| 短休息结束 | 专注时间到 🍅 | 开始第 N+1 个番茄 |
| 长休息结束 | 新的循环 ✨ | 休息完毕，重新开始 |

### 托盘图标

- **运行中：** 标题栏显示剩余时间，如 `19:35`
- **暂停中：** 显示 `⏸ 25:00`
- **右键菜单：** 开始/暂停 · 跳过当前段 · 退出

### 数据持久化

今日完成番茄数使用 `localStorage` 存储，跨重启保留，无需数据库。

---

## 六、边界处理

- **段结束自动推进：** 每段结束后自动发通知并立即开始下一段（休息/专注均自动开始），无需用户点击
- **窗口关闭：** 关闭主窗口不退出应用，仅隐藏窗口，保持托盘运行
- **系统休眠唤醒：** 唤醒后重新计算剩余时间，若当前段已超时则跳到下一段（只跳一步，不连续推进多段），并发通知
- **多实例：** `main.js` 使用 `app.requestSingleInstanceLock()` 防止重复启动
