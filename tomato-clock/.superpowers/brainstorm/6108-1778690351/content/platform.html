番茄钟用什么技术平台来做？

选择决定了开发方式、分发方式和最终体验

A

Electron（推荐）

用 HTML/CSS/JS 构建，打包成 .app 安装包。开发体验好，生态丰富，可跨平台。

优点

  • Web 技术栈，开发快
  • 系统通知、托盘菜单等原生集成
  • 打包成独立 .app

缺点

  • 打包体积较大（约 100MB）
B

Tauri

Web 前端 + Rust 后端，体积极小。相比 Electron 更轻量，但需要安装 Rust 工具链。

优点

  • 打包体积小（约 3-10MB）
  • 性能更好，内存占用低

缺点

  • 需要 Rust 环境，配置稍复杂
C

单文件 HTML

一个 HTML 文件，双击用浏览器打开即可运行。零依赖，最简单，但功能受限（无系统通知、无托盘图标）。

优点

  • 零安装，双击即用
  • 开发极快，适合快速验证

缺点

  • 不是真正的"桌面应用"
  • 无系统级通知和托盘菜单
